⚡ Как установить и запустить (одна команда)

   Скопируйте эту строку и вставьте в терминал вашего сервера:

   bash <(curl -sL https://raw.githubusercontent.com/Pykucyka/remnasetup/main/remnawave-manager.sh)

   Или, если хотите клонировать репозиторий и запускать локально:

   git clone https://github.com/Pykucyka/remnasetup.git
   cd remnasetup
   bash remnawave-manager.sh

══════════════════════════════════════════════════════════════

📋 Главное меню (после запуска)

   Выберите раздел:
   1) 🖥️  Панель Remnawave (установка, обновление, бэкапы)
   2) 📡 RemnaNode (установка, логи, удаление)
   3) 📄 Страница подписок (установка, обновление, управление)
   0) 🚪 Выход

   В каждом разделе — полный набор действий с подсказками,
   прогресс-барами и проверкой ошибок.

══════════════════════════════════════════════════════════════

✨ Что умеет скрипт

   🖥️  Панель Remnawave
      — Установка с нуля (клонирование, .env, Docker Compose)
      — Просмотр и редактирование переменных окружения
      — Обновление до последней версии (git pull + пересборка)
      — Логи всех сервисов в реальном времени
      — Статус контейнеров (docker-compose ps)
      — Проверка текущей версии
      — Полное удаление с очисткой данных
      — Создание резервной копии (конфиги + дамп БД)
      — Восстановление из архива с выбором файла

   📡 RemnaNode
      — Локальная установка ноды и подключение к панели
      — Просмотр логов через journalctl
      — Статус сервиса
      — Проверка версии
      — Удаление ноды с конфигурацией

   📄 Страница подписок Remnawave
      — Установка из официального репозитория
      — Настройка домена, API‑ключа, секрета
      — Обновление, логи, статус, удаление

   🎨 Интерфейс
      — Анимированные спиннеры и линейные прогресс-бары
      — Понятные подсказки и сообщения об ошибках
      — Обработка прерываний с указанием строки сбоя
      — Автоустановка Docker и docker-compose при необходимости

══════════════════════════════════════════════════════════════

🔧 Требования

   — Ubuntu 20.04 / 22.04 / 24.04 (Debian 11/12 тоже подойдут)
   — Доступ в интернет
   — Права root (рекомендуются, особенно для установки ноды)
   — Docker и docker-compose — установятся автоматически, если их нет

══════════════════════════════════════════════════════════════

🛡️ Безопасность

   — Все пароли и ключи генерируются случайно
   — Файл .env содержит конфиденциальные данные — не публикуйте его
   — Используются только официальные репозитории Remnawave
   — При удалении контейнеры и тома Docker полностью очищаются
