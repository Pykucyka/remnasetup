## 📦 Быстрый старт (одна команда)

```bash
bash <(curl -sL https://raw.githubusercontent.com/Pykucyka/remnasetup/main/remnawave-manager.sh)
```

или классическим способом:

```bash
git clone https://github.com/Pykucyka/remnasetup.git
cd remnasetup
chmod +x remnawave-manager.sh
./remnawave-manager.sh
```

После запуска вы попадёте в главное меню с тремя разделами.  
Всё управляется через цифровые пункты – скрипт проведёт вас по шагам, показывая прогресс-бары и подсказки.

---

## 🧩 Что умеет скрипт

### 🖥️ Панель Remnawave
- Установка с нуля (клонирование официального репозитория, настройка `.env`, запуск Docker Compose)
- Просмотр и редактирование переменных окружения (`.env`)
- Обновление панели (`git pull` + пересборка контейнеров)
- Логи всех сервисов в реальном времени
- Статус контейнеров (`docker-compose ps`)
- Просмотр текущей версии
- Полное удаление с очисткой данных и томов
- 💾 Создание резервной копии (конфиги + дамп БД PostgreSQL)
- 📥 Восстановление из резервной копии (выбор архива, замена конфигов, импорт дампа)

### 📡 RemnaNode
- Установка локальной ноды с автоматической привязкой к панели
- Просмотр логов (`journalctl -f`)
- Статус сервиса
- Версия ноды
- Удаление ноды с полной очисткой

### 📄 Страница подписок Remnawave
- Установка из официального репозитория
- Настройка домена, API‑URL панели и секретного ключа
- Обновление (`git pull` + пересборка)
- Логи и статус контейнеров
- Удаление

### 🎨 Дополнительно
- Красивый цветной интерфейс с анимированными спиннерами и линейными прогресс-барами
- Обработка ошибок с указанием строки и кода
- Автоматическая установка Docker и docker-compose, если они отсутствуют
- Подсказки на каждом шаге

## 🔧 Системные требования

- Ubuntu 20.04 / 22.04 / 24.04 (Debian 11/12 тоже поддерживаются)
- Доступ в интернет
- Права **root** (рекомендуются, особенно для установки ноды; для панели достаточно обычного пользователя с Docker)
- Docker и docker-compose – установятся автоматически при необходимости

---

## 🛡️ Безопасность

- Все пароли и ключи генерируются случайно с помощью `openssl rand`
- Файл `.env` содержит конфиденциальные данные – не публикуйте его
- Используются только официальные репозитории Remnawave
- При удалении панели или подписки все тома Docker уничтожаются (рекомендуется делать бэкап заранее)
