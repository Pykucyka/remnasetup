══════════════════════════════════════════════════════════════
🚀 БЫСТРЫЙ СТАРТ (две команды на выбор)
══════════════════════════════════════════════════════════════

   1. Запуск без клонирования (прямо с GitHub):
      Вставьте эту строку в терминал сервера:

      bash <(curl -sL https://raw.githubusercontent.com/Pykucyka/remnasetup/main/remnawave-manager.sh)

   2. Классическое клонирование:
      git clone https://github.com/Pykucyka/remnasetup.git
      cd remnasetup
      chmod +x remnawave-manager.sh
      ./remnawave-manager.sh

   После запуска вы попадёте в главное меню. Всё остальное
   делается выбором пунктов — скрипт проведёт вас по шагам.

══════════════════════════════════════════════════════════════
⚙️  ЧТО СКРИПТ ДЕЛАЕТ ПРИ ПЕРВОМ ЗАПУСКЕ
══════════════════════════════════════════════════════════════

   • Проверяет, установлен ли Docker и docker‑compose.
     Если нет – автоматически скачивает и настраивает их.
   • Проверяет наличие curl, git и других зависимостей.
   • Предупреждает, если запущен не от root (для установки
     ноды права root обязательны, для панели – достаточно
     прав обычного пользователя с Docker).

📋 ГЛАВНОЕ МЕНЮ
══════════════════════════════════════════════════════════════

   После запуска вы видите три основных раздела:

   1) 🖥️  Панель Remnawave — установка, настройка,
      обновление, бэкап и восстановление.
   2) 📡 RemnaNode — управление локальной нодой.
   3) 📄 Страница подписок — установка и обслуживание
      страницы для клиентов.

   Выберите нужный раздел, и откроется подробное меню со
   всеми действиями.

══════════════════════════════════════════════════════════════
🖥️  ПАНЕЛЬ REMNAWAVE – ВСЕ ДЕЙСТВИЯ
══════════════════════════════════════════════════════════════

   1) Установить / Настроить заново
      • Клонирует официальный репозиторий Remnawave.
      • Запрашивает домен панели и пароль администратора.
      • Генерирует безопасные ключи JWT, API, пароли БД.
      • Записывает .env и запускает контейнеры через Docker.
      • После установки показывает адрес, логин и пароль.

   2) Просмотреть .env
      • Выводит на экран содержимое файла .env, чтобы вы
        могли проверить параметры.

   3) Редактировать .env
      • Открывает редактор nano. После сохранения изменений
        рекомендуется перезапустить панель (пункт 4).

   4) Обновить панель
      • Делает git pull из репозитория Remnawave.
      • Останавливает, пересобирает и запускает контейнеры.
      • Все настройки (включая .env) сохраняются.

   5) Логи сервисов
      • Показывает объединённые логи всех контейнеров в
        реальном времени. Для выхода нажмите Ctrl+C.

   6) Статус контейнеров
      • Показывает состояние всех Docker‑сервисов панели
        (работает / остановлен).

   7) Версия панели
      • Определяет установленную версию через git describe.

   8) Удалить панель
      • Останавливает контейнеры, удаляет тома (включая БД)
        и полностью удаляет папку с панелью. Требуется
        подтверждение.

   9) 💾 Создать резервную копию
      • Создаёт полный бэкап: сохраняет .env, docker‑compose.yml
        и дамп базы данных PostgreSQL.
      • Упаковывает всё в архив с датой и временем.
        Файл кладётся в папку backups рядом со скриптом.
      • Процесс сопровождается прогресс‑баром.

  10) 📥 Восстановить из резервной копии
      • Показывает список доступных архивов в папке backups.
      • Выберите нужный архив (по имени файла).
      • Скрипт распакует архив, остановит текущую панель,
        заменит .env и docker‑compose.yml, запустит панель и
        импортирует дамп базы данных обратно в PostgreSQL.
      • После завершения панель сразу готова к работе.

══════════════════════════════════════════════════════════════
📡 REMNANODE – УПРАВЛЕНИЕ НОДОЙ
══════════════════════════════════════════════════════════════

   1) Установить ноду
      • Требует, чтобы панель была уже установлена и в .env
        были указаны API_KEY и DOMAIN.
      • Запускает официальный установщик RemnaNode, передавая
        ему ключ API и домен панели.
      • После завершения нода сразу подключается к панели.

   2) Логи ноды
      • Просмотр логов сервиса remnanode (journalctl -f).
        Для выхода Ctrl+C.

   3) Статус ноды
      • Проверяет, активен ли сервис remnanode, и выводит
        подробный статус.

   4) Версия ноды
      • Показывает версию установленного remnanode, если
        команда доступна, либо сообщает, что версия не определена.

   5) Удалить ноду
      • Останавливает и отключает сервис, удаляет все файлы.
        Требуется подтверждение.

══════════════════════════════════════════════════════════════
📄 СТРАНИЦА ПОДПИСОК – УПРАВЛЕНИЕ SUBSCRIPTION PAGE
══════════════════════════════════════════════════════════════

   1) Установить страницу подписок
      • Клонирует официальный репозиторий subscription-page.
      • Запрашивает домен страницы (например, sub.example.com).
      • Автоматически подставляет API‑URL панели, если она
        установлена (можно изменить вручную).
      • Генерирует уникальный SUB_SECRET.
      • Запускает Docker‑контейнеры. После установки выводит
        адрес новой страницы.

   2) Обновить страницу подписок
      • Выполняет git pull и пересобирает контейнеры.

   3) Логи страницы подписок
      • Показывает логи контейнеров в реальном времени.

   4) Статус контейнеров
      • Показывает состояние Docker‑сервисов страницы подписок.

   5) Удалить страницу подписок
      • Останавливает контейнеры и удаляет папку. Требуется
        подтверждение.

══════════════════════════════════════════════════════════════
💡 ВАЖНЫЕ СОВЕТЫ
══════════════════════════════════════════════════════════════

   • Всегда делайте бэкап перед обновлением или удалением.
     Резервные копии находятся в папке backups и называются
     remnawave_backup_YYYYMMDD_HHMMSS.tar.gz.
   • При восстановлении сначала убедитесь, что архив не
     повреждён и содержит файлы .env, docker‑compose.yml и
     remnawave_db.sql.
   • Если восстановление БД не удалось, можно импортировать
     дамп вручную командой, которую покажет скрипт.
   • Для установки ноды обязательно иметь root‑права, иначе
     сервис systemd не создастся.
   • Страницу подписок можно запускать на том же сервере, где
     панель, или на отдельном – достаточно указать API‑URL.

🛡️ БЕЗОПАСНОСТЬ
══════════════════════════════════════════════════════════════

   • Все ключи генерируются с помощью openssl rand.
   • Файл .env не предназначен для публикации — храните его
     в безопасном месте.
   • Скрипт работает только с официальными репозиториями
     Remnawave на GitHub.
   • При удалении панели все тома Docker уничтожаются, поэтому
     без бэкапа данные потеряются.
